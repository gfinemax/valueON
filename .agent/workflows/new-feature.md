# New Feature Kick-off

Use this workflow to start a new feature systematically.

## Step 1: Requirements Analysis
1. Ask the user for the feature description if not already provided.
2. Update `PRD.md`: Add a new section for this feature with "Background" and "Requirements".

## Step 2: Planning
1. Create or update `implementation_plan.md`.
2. Break down the work into technical tasks (Components, API, DB).
3. Update `task.md` with a checklist for this feature.

## Step 3: User Review
1. Review the plan with the user before writing code.
2. Ask: "Does this plan look correct?"

## Step 4: Ready to Code
1. Once approved, verify the plan and start the first task.
