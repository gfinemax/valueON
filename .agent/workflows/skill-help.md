# 🤖 AI 스킬(워크플로우) 도움말

이 문서는 AI 에이전트에게 반복적인 작업을 효율적으로 지시하기 위한 '스킬(워크플로우)'에 대해 설명합니다.

## 1. 스킬(Skill)이란?
*   **비유:** 요리사에게 주는 **'비법 레시피'**
*   **개념:** AI가 수행해야 할 작업의 순처와 방법을 미리 정의해 둔 마크다운(`*.md`) 파일입니다.
*   **효과:** 매번 길게 설명할 필요 없이 "이 기능 마무리해줘", "배포 준비해줘" 한마디로 복잡한 작업을 처리할 수 있습니다.

## 2. 스킬 생성 방법
프로젝트 루트의 `.agent/workflows/` 폴더에 `.md` 파일을 생성하면 됩니다.

**예시 기본 구조:**
```markdown
---
description: 이 스킬에 대한 간단한 설명
---
# 스킬 이름

1. 첫 번째로 할 일
2. 두 번째로 할 일
   - 구체적인 지시 사항
```

## 3. 다른 프로젝트에서 재사용하기
스킬 파일은 프로젝트 독립적입니다. 내 컴퓨터의 다른 프로젝트에서도 똑같이 쓰고 싶다면:
1.  **파일 복사:** `.agent/workflows/` 안에 있는 `.md` 파일을 복사합니다.
2.  **붙여넣기:** 새 프로젝트의 동일한 경로(`.agent/workflows/`)에 붙여넣습니다.
3.  **팁:** 자주 쓰는 스킬들은 `C:\Antigravity_Skills` 같은 별도 폴더에 백업해 두고, 필요할 때마다 복사해서 쓰면 편리합니다.

## 4. 다른 AI 에이전트와의 호환성
이 스킬 파일은 **'사람의 말(자연어)'**로 작성된 작업 지시서이므로, 안티그라비티 외 다른 AI와도 호환됩니다.

*   **클로드(Claude):** [Project Knowledge]에 파일을 업로드하고 "이 파일대로 해줘"라고 지시.
*   **커서(Cursor):** 채팅창에 내용을 붙여넣거나 `.cursorrules`로 활용.
*   **ChatGPT:** [Custom Instructions]에 내용을 저장하여 활용.

> **주의:** `// turbo` 같은 안티그라비티 전용 명령어(자동 실행)는 다른 AI에서는 무시되거나 단순 텍스트로 인식될 수 있습니다. (기능 수행에는 지장 없음)

## 5. 사용 가능한 스킬 및 간단 명령어

| 스킬 명 | 명령어 | 자연어 실행 예시 | 설명 |
| :--- | :--- | :--- | :--- |
| **버그 수정 마법사** | `/bug-fix` | "이 버그 고쳐줘" | 버그 수정 후 빌드 검증, 문서 확인, 커밋까지 자동 처리 |
| **배포 준비** | `/deploy-prep` | "배포 준비해줘" | 타입 체크, 린트 검사, 콘솔 로그 정리, 빌드 테스트 일괄 수행 |
| **기능 마무리** | `/feature-wrapup` | "이번 기능 마무리해줘" | 개발 완료 후 PRD 및 사용자 매뉴얼 업데이트, Git 커밋 |
| **새 기능 시작** | `/new-feature` | "새 기능 시작하자" | 새 기능 요구사항 분석, PRD 업데이트, Task 목록 생성 |
| **코드 리팩토링** | `/refactor-check` | "코드 정리해줘" | 불필요한 import 제거, 코드 구조 개선 및 주석 정리 |
| **스킬 도움말** | `/skill-help` | "스킬 도움말 보여줘" | 스킬의 개념, 생성 및 활용 방법에 대한 가이드 문서 |

> **참고:** 새로운 스킬이 추가되면 이 표에 해당 스킬의 정보를 함께 업데이트해 주세요.
